# 오목 게임 — 기능 명세서 (Functional Specification)

> 범위: **2인 로컬 대전 오목** / 단일 HTML 파일 / 오프라인 실행

---

## 1. 개요

한 대의 PC에서 두 사람이 번갈아 마우스로 돌을 놓아 대국하는 오목 게임.
흑(선)과 백이 교대로 착수하며, 먼저 가로·세로·대각선으로 돌 5개를 연속으로 놓는 쪽이 승리한다.

---

## 2. 용어 정의

| 용어 | 정의 |
|---|---|
| 착수(着手) | 빈 교차점에 돌을 놓는 행위 |
| 흑선 | 흑이 먼저 두는 규칙 |
| 5목(五目) | 같은 색 돌 5개가 한 방향으로 연속된 상태 = 승리 조건 |
| 장목 | 6개 이상 연속. 본 게임(자유룰)에서는 승리로 인정 |
| 화점 | 보드 위 기준점(검은 점). 15x15에서는 천원 포함 5개 |
| 천원 | 보드 정중앙 교차점 (8,8) |

---

## 3. 기능 요구사항 (Functional Requirements)

| ID | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| **FR-01** | 보드 표시 | 15×15 격자, 화점 5개, 좌표선을 캔버스에 렌더링 | **Must (MVP)** |
| **FR-02** | 착수 | 빈 교차점 클릭 시 현재 차례의 돌을 놓음. 이미 돌이 있거나 보드 밖이면 무시 | **Must (MVP)** |
| **FR-03** | 턴 관리 | 착수 성공 시 흑↔백 교대. 현재 차례를 상단 상태바에 표시 | **Must (MVP)** |
| **FR-04** | 승리 판정 | 착수 직후 4방향 5목 검사. 충족 시 게임 종료 | **Must (MVP)** |
| **FR-05** | 승리 연출 | 승자 메시지 표시 + 승리한 5개 돌 강조(라인/하이라이트). 종료 후 착수 차단 | **Must (MVP)** |
| **FR-06** | 재시작 | "다시 시작" 버튼으로 보드·상태 초기화 | **Must (MVP)** |
| **FR-07** | 무승부 | 빈 칸 없이 보드가 가득 차고 승자 없으면 무승부 처리 | Should |
| **FR-08** | 마지막 수 표시 | 가장 최근 착수한 돌에 표식(점/테두리)으로 위치 강조 | Should |
| **FR-09** | 무르기(Undo) | "무르기" 버튼으로 직전 착수 1수 취소(연속 가능) | Should |
| **FR-10** | 수순 표시 | 각 돌에 착수 순번 숫자 표시(토글) | Could |
| **FR-11** | 호버 미리보기 | 마우스 올린 교차점에 반투명 돌 미리보기 | Could |
| **FR-12** | 사운드 | 착수/승리 효과음 | Won't (이번 범위 제외) |

---

## 4. 적용 게임 규칙

| 항목 | 정책 |
|---|---|
| 보드 크기 | 15 × 15 (표준) |
| 선후 | 흑선 (흑이 먼저) |
| 금수(禁手) | **없음 — 자유 오목룰**. 3·3, 4·4, 장목 금지 규칙 미적용 (가장 빠른 구현 우선) |
| 승리 조건 | 같은 색 5개 이상 연속(가로/세로/대각) |
| 장목 | 승리로 인정 |
| 무승부 | 보드 가득 + 승자 없음 |

> 렌주룰(금수)은 향후 확장 항목. 이번 범위에서는 단순·명확한 자유룰을 채택한다.

## 5. UI 요구사항

### 5.1 화면 구성 (와이어프레임)

```
┌──────────────────────────────────────┐
│  오목 (Omok)        ● 흑 차례          │  ← 상태바: 제목 + 현재 차례 표시
├──────────────────────────────────────┤
│                                      │
│        15 x 15 바둑판 (Canvas)         │  ← 메인 보드
│        · 교차점 클릭으로 착수            │
│        · 화점/마지막 수 표시            │
│                                      │
├──────────────────────────────────────┤
│   [ 다시 시작 ]      [ 무르기 ]         │  ← 컨트롤 버튼
└──────────────────────────────────────┘
```

### 5.2 상호작용

| 입력 | 결과 |
|---|---|
| 보드 교차점 클릭 | 유효하면 착수, 무효(점유/범위밖/게임종료)면 무반응 |
| "다시 시작" 클릭 | 보드 초기화, 흑 차례부터 재시작 |
| "무르기" 클릭 | 직전 1수 취소(기록 없으면 무반응) |
| 마우스 이동(Could) | 호버 교차점 미리보기 |

### 5.3 상태 표시
- 진행 중: `● 흑 차례` / `○ 백 차례`
- 종료: `● 흑 승리!` / `○ 백 승리!` / `무승부`

## 6. 게임 상태 정의 (State Machine)

```
        ┌─────────┐  착수(승/무 아님)   ┌─────────┐
        │ PLAYING │ ◄─────────────────► │ PLAYING │ (차례 교대)
        └────┬────┘                     └─────────┘
       5목 달성│ │보드 가득
             ▼ ▼
        ┌─────┐ ┌──────┐
        │ WIN │ │ DRAW │   ← 착수 차단(입력 무시)
        └──┬──┘ └──┬───┘
           │ 다시시작 │
           ▼        ▼
        ┌─────────────┐
        │  PLAYING    │ (초기 상태로)
        └─────────────┘
```

| 상태 | 의미 | 착수 허용 |
|---|---|---|
| `playing` | 대국 진행 중 | O |
| `win` | 한쪽 5목 달성 | X |
| `draw` | 보드 가득, 승자 없음 | X |

## 7. 비기능 요구사항 (Non-Functional)

| 항목 | 요구 |
|---|---|
| 실행 | `index.html` 더블클릭으로 브라우저에서 즉시 실행 (`file://`) |
| 의존성 | 외부 라이브러리·네트워크 불필요 |
| 성능 | 착수→화면 반영 즉각(체감 지연 없음). 5목 판정 O(1)에 가까움(국소 검사) |
| 호환성 | 최신 Chrome/Edge 데스크톱 |
| 코드 | 주석은 한국어. 모듈 경계는 ARCHITECTURE.md 준수 |

---

## 8. 인수 기준 (Acceptance Criteria) — MVP

- [ ] 15×15 보드와 화점이 정확히 그려진다.
- [ ] 빈 교차점 클릭 시 현재 차례 돌이 놓이고, 차례가 교대된다.
- [ ] 점유된 칸·보드 밖 클릭은 무시된다.
- [ ] 가로/세로/양 대각선 어느 방향이든 5목이면 즉시 승리 처리된다.
- [ ] 승리 후에는 더 이상 돌을 놓을 수 없고, 승자가 표시된다.
- [ ] "다시 시작"으로 초기 상태(흑 차례)로 되돌아간다.
